# Use cases / сценарии

> Этап 2. Сценарии использования. Фокус на альтернативных потоках.

## UC-1: Запись на заезд (себя и гостей)

- **Актёр:** Клиент
- **Предусловия:** Клиент авторизован; есть слот со свободными картами.
- **Триггер:** Клиент хочет забронировать карт.

> **R-013:** Собираем только количество мест и тип экипировки.
> **R-022 (Идемпотентность):** Обязательный `Idempotency-Key` для защиты от сетевых дублей.

### Основной поток
1. Клиент открывает список заездов.
2. Открывает карточку заезда (конфигурация трассы, маршал, цена, свободные карты).
3. Указывает число мест (не более 4 на бронь).
4. Выбирает экипировку для каждого: своя (шлем, подшлемник) или прокат.
5. Подтверждает запись (передаёт `Idempotency-Key`).
6. Сервер проверяет лимиты: `min(free_karts, track.capacity, 4)`. Для новичковой трассы потолок — 8 человек. Проверяется фонд шлемов.
7. Создаётся бронь; счётчик свободных картов уменьшается. Запись видна в «Мои бронированиях».

### Альтернативные потоки / Ошибки
- **A1. Своя экипировка:** Фонд проката не уменьшается, занимается только карт.
- **E1. Нехватка картов:** Запрошено больше картов, чем есть → отказ, показ актуального остатка (`slot_full`).
- **E3. Гонка запросов:** Другой клиент занял карт долей секунды раньше → отказ без овербукинга.
- **E4. Сбой сети:** Мутация офлайн запрещена. При обрыве повтор отправляется с тем же `Idempotency-Key`.
- **E5. Заезд отменён из-за непогоды:** Слот получает код `slot_cancelled`. Бронь отклоняется.

## UC-2: Отмена записи

- **Актёр:** Клиент
- **Предусловия:** Есть предстоящая бронь.
- **Триггер:** Клиент отменяет участие.

### Основной поток
1. Клиент нажимает «Отменить» (отмена всей брони целиком, включая все карты).
2. Сервер проверяет время до старта.
3. **Ранняя отмена (≥1 ч до старта):** Бронь отменяется; карты и экипировка возвращаются в свободный пул.

### Альтернативные потоки / Ошибки
- **A1. Поздняя отмена (<1 ч до старта, например за 10 минут):** Запись получает статус `late_cancel`. Карты НЕ освобождаются в расписании (штрафов нет, но блокируется пустая перепродажа).
- **E1. Заезд уже начался (`slot_started`):** Кнопка отмены недоступна.

## UC-3: Просмотр и фильтрация расписания

- **Актёр:** Клиент
- **Предусловия:** Авторизация пройдена.

### Основной поток
1. По умолчанию открывается список заездов на ближайшие 7 дней.
2. Слоты, где `free_karts = 0`, помечаются «Мест нет» (CTA заблокирована).
3. Клиент фильтрует по: конфигурации трассы (длинная/короткая), маршалу, дате.
4. Список обновляется по условиям И/ИЛИ (группы внутри).
