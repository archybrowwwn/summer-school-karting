# Нефункциональные требования

> Этап 2. Как система должна работать: производительность, надёжность, безопасность.
> **Скоуп:** клиентское мобильное приложение.

| ID | Категория | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- | :-- |
| NFR-1 | Удобство | Интерфейс — mobile-first, рассчитан на использование с телефона (на улице у трассы, при ярком свете): высокий контраст, крупные элементы. | Высокий | Бриф |
| NFR-2 | Удобство | Запись на заезд выполняется за минимальное число шагов (≤3 экранов). | Высокий | MVP-подход |
| NFR-4 | Платформа | Клиент — нативное/гибридное приложение для iOS и Android. Поддержка системного push, хранение токена в Keychain/Keystore. | Must | R-002 |
| NFR-7 | Нагрузка | Приложение выдерживает пиковую нагрузку в выходные (когда начинается «хаос» с записями). | Высокий | Бриф |
| NFR-8 | Надёжность | **Критическое:** При одновременной записи исключаются двойные брони (на один карт не могут записаться двое). Синхронизация транзакций на стороне бэкенда. | Must | Бриф (главная боль) |
| NFR-11 | Безопасность | ПДн (телефон, имя) защищены от несанкционированного доступа. | Высокий | NFR-11 |
| NFR-15 | Ограничения | Жёсткий срок запуска — старт высокого сезона (~2 месяца). | Must | Бриф |
| NFR-17 | Уведомления | MVP-канал: системный push. Напоминания отправляются за 24 ч и за 2 ч до старта заезда (тайминг берётся из ответа API). | Средний | Бриф; R-006 |
| NFR-18 | Сессия | Авторизация через JWT (access/refresh). Обмен по TLS (HTTPS, TLS 1.2+). При 401 — автообновление токена или перевод на экран входа. | Высокий | R-016 |
| NFR-19 | Антифрод OTP| Отправка SMS ограничена rate-limit (1 запрос в 30-60с, макс. 5 в час). | Высокий | R-016 |
| NFR-22 | Нагрузка | Сценарий нагрузочного теста: N клиентов одновременно бронируют последний свободный карт; подтверждается ровно 1 бронь, остальные получают `slot_full`. | Высокий | NFR-8 |
| NFR-24 | Offline | Просмотр расписания доступен офлайн из кэша. Создание/отмена брони офлайн запрещены. Паттерн `Idempotency-Key` для исключения дублей при сбоях сети. | Средний | R-020 |
| NFR-26 | Карты | Интеграция Яндекс.Карт для отображения адреса центра. Ключ в конфиге окружения. При отказе API карт — fallback режим (только текст адреса). | Средний | R-011 |
