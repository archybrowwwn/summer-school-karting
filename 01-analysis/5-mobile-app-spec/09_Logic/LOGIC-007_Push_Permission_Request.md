# Запрос push-разрешения

**ID:** LOGIC-007  
**Тип:** Логика  
**Домен:** 09. Логики  
**Приоритет:** Medium  
**Статус:** Черновик  

## Обзор

Системный запрос разрешения на push-уведомления показывается **только после первой успешной брони**.

### Логика показа
- Первый раз — после успешного оформления брони (BS-002).
- В дальнейшем — не показывать повторно.
- Пользователь может позже включить уведомления в настройках профиля.

### User Story
> Как новый клиент, я хочу получать напоминания о предстоящем заезде,
> но не хочу, чтобы меня сразу просили включить уведомления при первом запуске.

## Точки применения
- [BS-002 Подтверждение брони](../BS-002-booking-success.md) — после успешной записи
- Настройки профиля (SCR-007) — ручное управление
