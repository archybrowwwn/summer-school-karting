## 1. Назначение

**«Апекс»** — клиентское мобильное приложение для самостоятельной записи на групповые заезды в уличном картинг-центре «Апекс». Заменяет ручную запись через Telegram и бумажную/маркерную доску, устраняя двойные брони, путаницу с картами и простой инвентаря.

**Скоуп приложения — только роль «Клиент».** Маршал-инструктор и Владелец (Денис) работают через существующую инфраструктуру/админку и в приложение **не входят**. Справочные данные (заезды/слоты, конфигурации трасс, маршалы, тарифы проката экипировки, свободный фонд) приложение получает из API в режиме **read-only**; оплата — **офлайн** (наличные / перевод на карту), приложение лишь показывает итоговую цену и фиксирует запись. Гарантия отсутствия двойных броней обеспечивается на стороне бэкенда (атомарная проверка свободных картов).

## 2. Глоссарий и роли

| Термин | Значение |
|--------|----------|
| **Заезд / Слот** | Конкретный заезд группы по трассе: дата, время старта, конфигурация трассы, маршал-инструктор, всего/свободно картов, статус, базовая цена + тариф проката экипировки (при выборе). Один заезд идёт 15–20 мин + инструктаж. |
| **Конфигурация трассы** | Вариант трассы — **короткая** (попроще для новичков, часть картов с ограничением скорости, max ~8 человек) и **длинная** (для опытных, до 14 картов). |
| **Маршал-инструктор** | Проводит брифинг перед стартом, следит за трассой и безопасностью во время заезда. Сейчас 5 постоянных + сезонные подработчики летом. |
| **Экипировка** | Шлем + подшлемник. **Своя** (клиент привозит с собой) или **прокатная** (из фонда центра, есть тариф). |
| **Запись / Бронь** | Резервирование клиентом одного или нескольких мест на заезд. Для каждого места указывается вариант экипировки (своя / прокатная). |
| **Ранняя отмена** | Отмена клиентом не позднее чем за 30 минут до старта заезда → место(а) возвращаются в слот, бронь удаляется. |
| **Поздняя отмена** | Отмена клиентом менее чем за 30 минут до старта → бронь помечается «поздняя отмена», место **не** освобождается (карт не простаивает впустую), штрафов нет (чтобы не злить клиентов). |
| **Отмена центром** | Заезд отменяет организатор или по погоде (дождь, гололёд — трасса уличная). Бронь клиента получает статус **«Отменён центром»** с указанием причины; приходит push-уведомление. Повторная запись на отменённый слот запрещена. |

**Роль приложения:** **Клиент** — просматривает и фильтрует доступные заезды, записывается (выбирает число мест и вариант экипировки для каждого), отменяет свои записи (с соблюдением правила 30 мин), получает push-уведомления (напоминания + отмены центром), после заезда может поставить оценку маршалу.

> **Принцип абстракции.** В фича-листе **не привязываемся к конкретным числам** (размер прокатного фонда экипировки, точные потолки конфигураций, цены, длительность). Все лимиты и цены — параметры, приходящие из данных/конфигурации заезда и конфигурации трассы.
>
> **Раздельная модель доступности (места ≠ прокатный фонд экипировки).** Места (карты) и прокатный фонд экипировки считаются **независимо**:
> - **Лимит мест группы:** `max_seats = min(free_karts, config.capacity_cap)` — значения из данных заезда/конфигурации (для короткой обычно cap=8).
> - **Лимит прокатного фонда:** `rental_count ≤ free_rental_gear` (фонд экипировки).
>
> «Своя экипировка» занимает место в группе, но **не** уменьшает прокатный фонд; «Прокатная» — занимает место **и** уменьшает прокатный фонд. Доступность мест считается отдельно от доступности прокатной экипировки — это два разных лимита.
>
> **Лимит на одного клиента:** до 2 мест на один заезд (себя + 1 гость) — согласовано как разумное ограничение для старта, чтобы не усложнять брифинг и контроль.

## 5. Детализация по экранам

### SCR-001 · Регистрация / Вход

- **Назначение:** минимальный порог входа — регистрация и повторный вход по телефону (без сложного пароля).
- **Зона:** НЗ.
- **Доступные функции:**
  - Ввод имени и номера телефона.
  - Подтверждение входа (код из SMS — детали на стороне бэкенда).
  - Повторный вход по номеру телефона (если уже регистрировался).
- **Ключевые элементы:** поле «Имя», поле «Телефон», кнопка «Продолжить».
- **Бизнес-правила/валидации:** валидация формата имени (кириллица/латиница, 2–30 символов) и телефона (`^9\d{9}$` или международный); отсутствие сложного пароля (NFR-3); ≤ минимально необходимых полей (NFR-2).
- **Требования:** FR-1, FR-2 / US-1 / NFR-3.

### SCR-002 · Список заездов

- **Назначение:** главный экран — список доступных заездов; точка входа в запись.
- **Зона:** АЗ.
- **Доступные функции:**
  - Просмотр списка заездов (по умолчанию — **ближайшие 7 дней**, `only_available=false`; больший период — фильтром дат).
  - Открыть шторку фильтров [BS-001](#bs-001--фильтры).
  - Переход в карточку заезда [SCR-003](#scr-003--карточка-заезда).
  - Pull-to-refresh для обновления доступности.
  - Переходы в «Мои записи» и «Профиль».
- **Ключевые элементы:** карточка заезда (дата/время старта, конфигурация трассы и тип, маршал, цена от..., всего/свободно картов, индикатор доступности прокатной экипировки); индикатор активных фильтров; статус заезда (идёт запись / отменён центром / completed).
- **Бизнес-правила/валидации:** список показывает заезды на **ближайшие 7 дней** (дефолт по R-027); заполненные (свободных картов нет) и отменённые заезды **не скрываются**, а помечаются соответственно («Мест нет» с неактивной CTA или «Отменён центром»); счётчик свободных картов отражает актуальные данные. По умолчанию `only_available=false`; скрыть заполненные можно фильтром «только со свободными местами» (BS-001). Если расписания на ближайшие дни нет — empty state «Пока нет доступных заездов».
- **Требования:** FR-9, FR-38 / UC-3, US-2, US-3 / NFR-6.

### BS-001 · Фильтры

- **Назначение:** уточнить список заездов под запрос клиента.
- **Зона:** АЗ.
- **Доступные функции:**
  - Фильтр по дате / периоду старта.
  - Фильтр по типу конфигурации трассы (короткая / длинная).
  - Фильтр «только со свободными местами».
  - Фильтр по маршалу-инструктору.
  - Применить / Сбросить фильтры.
- **Бизнес-правила/валидации:** фильтры комбинируются по «И»; при пустом результате — empty state с подсказкой изменить/сбросить фильтры.
- **Требования:** FR-38 / US-3.

### SCR-003 · Карточка заезда

- **Назначение:** показать все параметры заезда, чтобы клиент решил записаться.
- **Зона:** АЗ.
- **Доступные функции:**
  - Просмотр полных параметров заезда.
  - Переход к оформлению записи [SCR-004](#scr-004--оформление-записи).
  - Тап по блоку «Место сбора» → [BS-004](#bs-004--место-сбора).
- **Ключевые элементы:** дата/время старта, конфигурация трассы и её тип (короткая/длинная), **место сбора/брифинга** (адрес + пин на статичном превью Яндекс.Карты), маршал-инструктор (с фото если есть), базовая цена заезда, тариф проката экипировки, всего/свободно картов, доступность прокатного фонда экипировки, статус; кнопка «Записаться».
- **Бизнес-правила/валидации:** кнопка «Записаться» неактивна, если свободных картов нет или заезд отменён центром; место сбора — обязательный элемент; длительность ≈ 15–20 мин + инструктаж (берётся из данных конфигурации).
- **Требования:** FR-9a / US-4.

### SCR-004 · Оформление записи

- **Назначение:** собрать параметры брони и зафиксировать запись.
- **Зона:** АЗ.
- **Доступные функции:**
  - Выбор числа мест: 1 (себя) или 2 (себя + 1 гость).
  - Для каждого места — переключатель «Своя экипировка» / «Прокатная» (с отображением тарифа при выборе прокатной).
  - Просмотр итоговой цены (базовая × кол-во + прокат × кол-во прокатных).
  - Подтверждение записи → [BS-002](#bs-002--подтверждение-записи-вы-записаны).
- **Ключевые элементы:** счётчик/сегментированный контроль числа мест (1–2), карточки/переключатели экипировки по местам, блок итоговой цены, кнопка «Записаться».
- **Бизнес-правила/валидации:**
  - Места и прокатный фонд экипировки — **два независимых лимита**: `max_seats = min(free_karts, config.capacity_cap, 2)` для мест и `rental_count ≤ free_rental_gear` для проката — значения из данных заезда/конфигурации, не хардкодим.
  - Запись «своя экипировка» занимает место в группе, но **не** уменьшает прокатный фонд; «прокатная» — занимает место **и** уменьшает прокатный фонд (FR-14).
  - Запрет записи сверх лимита мест или свободного прокатного фонда экипировки (FR-13, FR-15).
  - Защита от двойной брони и овербукинга при параллельных записях (NFR-8) — обеспечивается бэкендом.
  - Обработка ошибок: нехватка мест (E1), нехватка прокатной экипировки с предложением уменьшить число прокатных / выбрать «свою» (E2), гонка запросов (E3), сетевой сбой (E4).
- **Требования:** FR-10–15, FR-30 / UC-1, US-5, US-6, US-7, US-8, US-11 / NFR-2, NFR-8.

### BS-002 · Подтверждение записи («Вы записаны»)

- **Назначение:** подтвердить успешную бронь и показать дальнейшие шаги.
- **Тип:** **Экран** (полноэкранный успех, RR-D03 аналог).
- **Зона:** АЗ.
- **Доступные функции:**
  - Просмотр сводки записи (заезд, число мест, экипировка по местам, итоговая цена).
  - Напоминание об офлайн-оплате (наличные / перевод на карту при приезде).
  - Переход в «Мои записи» [SCR-005](#scr-005--мои-записи) (primary).
  - «Готово» — возврат к списку «Заезды» [SCR-002](#scr-002--список-заездов) (secondary).
- **Бизнес-правила/валидации:** запись появляется в «Моих записях»; свободные карты заезда уменьшены. Свободного закрытия свайпом/бэкдропом нет — уход только по двум кнопкам. Приходит push-напоминание за N часов до старта (настраивается в инфраструктуре).
- **Требования:** FR-30 / US-5.

### SCR-005 · Мои записи

- **Назначение:** контроль предстоящих и прошедших заездов клиента.
- **Зона:** АЗ.
- **Доступные функции:**
  - Просмотр списка своих записей (предстоящие / прошедшие / отменённые).
  - Переход к деталям брони [SCR-006](#scr-006--детали-брони--отмена--оценка).
- **Ключевые элементы:** карточка записи (статус: предстоящая / прошедшая / отменена / отменена центром, параметры заезда, число мест, экипировка, цена).
- **Бизнес-правила/валидации:** клиент видит только свои записи (NFR-12); empty state, если записей нет; прошедшие записи позволяют поставить/посмотреть оценку маршалу.
- **Требования:** FR-35a / US-9 / NFR-12.

### SCR-006 · Детали брони + отмена + оценка

- **Назначение:** показать полную информацию о брони, дать отменить её (до старта) или поставить оценку маршалу (после заезда, если ещё не поставлена).
- **Зона:** АЗ.
- **Доступные функции:**
  - Просмотр деталей записи и статуса.
  - Запуск отмены → [BS-003](#bs-003--подтверждение-отмены) (только если заезд ещё не начался и статус позволяет).
  - Если заезд завершён и оценка ещё не поставлена — кнопка/блок «Поставить оценку маршалу» → [BS-005](#bs-005--оценка-маршала).
- **Ключевые элементы:** статус, дата/время, конфигурация трассы, **место сбора** (адрес + превью карты), маршал, число мест + экипировка по местам, цена; кнопки «Отменить» / «Поставить оценку».
- **Бизнес-правила/валидации:**
  - Кнопка «Отменить» доступна только до старта заезда и только для активных броней (E1).
  - Повторная отмена уже отменённой записи не выполняется (E2).
  - Оценка доступна только для прошедших заездов и только один раз (E3).
- **Требования:** FR-16, FR-17, FR-18, FR-20 / UC-2, US-10, US-12.

### BS-003 · Подтверждение отмены

- **Назначение:** объяснить последствия отмены (ранняя / поздняя) и подтвердить действие.
- **Зона:** АЗ.
- **Доступные функции:**
  - Просмотр правила 30 минут и текущего статуса (ранняя отмена / поздняя отмена).
  - Подтверждение / отказ от отмены.
- **Бизнес-правила/валидации:**
  - **Ранняя отмена** (≥ 30 мин до старта): места возвращаются в слот, бронь удаляется (FR-17).
  - **Поздняя отмена** (< 30 мин до старта): бронь помечается «поздняя отмена», место **не** освобождается, штрафов нет (FR-18).
  - Время отсечки вычисляется от времени старта заезда (берётся из данных слота).
- **Требования:** FR-17, FR-18 / US-10.

### BS-004 · Место сбора

- **Назначение:** показать точное место сбора/брифинга и как добраться.
- **Зона:** АЗ.
- **Доступные функции:**
  - Просмотр адреса центра и места сбора.
  - Статичный превью карты Яндекс с пином места сбора.
  - Кнопка «Открыть в Яндекс.Картах» (deeplink).
- **Бизнес-правила/валидации:** данные (адрес, координаты, описание) приходят из API слота/центра.
- **Требования:** FR-9a / US-4.

### SCR-007 · Профиль клиента

- **Назначение:** контактные данные, статус постоянного клиента, выход и удаление аккаунта.
- **Зона:** АЗ.
- **Доступные функции:**
  - Просмотр и редактирование имени и телефона (смена телефона — с подтверждением кодом из SMS).
  - Индикатор «Постоянный клиент» (если бэкенд отдаёт aggregate: кол-во заездов / статус).
  - Выход из аккаунта.
  - Удаление аккаунта (с обязательным подтверждением и предупреждением о последствиях для активных броней).
- **Бизнес-правила/валидации:** доступ только к собственным данным (NFR-11, NFR-12); при удалении аккаунта активные брони аннулируются и освобождают места, прошедшие — анонимизируются.
- **Требования:** FR-1, FR-2 / NFR-11, NFR-12.

### BS-005 · Оценка маршала

- **Назначение:** дать клиенту возможность оценить работу маршала после заезда (ключевой запрос заказчика для мотивации инструкторов).
- **Зона:** АЗ.
- **Доступные функции:**
  - Выбор 1–5 звёзд (с визуальной обратной связью).
  - Опциональный короткий комментарий (до 300 символов).
  - Отправка оценки (анонимно или с именем — по решению бэкенда).
- **Бизнес-правила/валидации:** оценка возможна только один раз на прошедшую бронь; после отправки — блокировка повторной отправки и отображение «Спасибо за оценку!».
- **Требования:** FR-20 / US-12.

## 6. Сквозные функции (не отдельные экраны)

- **Напоминания / уведомления** (FR-33, NFR-13): заблаговременное push-напоминание о предстоящей записи (за 24 ч и/или за 2 ч); уведомление об отмене заезда центром (с причиной). Канал в MVP — **системный push** (приложение регистрирует push-токен, доставку обеспечивает существующая инфраструктура). SMS / email / inbox — Phase 2.
- **Состояния экранов**: единый паттерн Loading (скелетон/шиммер) → Content → Empty (заглушка с подсказкой) → Error (с кнопкой «Обновить»). Применяется ко всем экранам со запросами.
- **NFR, влияющие на UI**: mobile-first для использования на улице/у трассы — крупные элементы, высокий контраст, удобные тапы даже в перчатках (NFR-1); запись ≤ 3–4 экранов до подтверждения (NFR-2); отклик списка и подтверждения < 2–3 с (NFR-6). Приложение — нативное/гибридное, распространяется через App Store / Google Play.
- **Push-разрешение**: при первом запуске или перед первой записью — запрос разрешения на push (см. [LOGIC-007](../09_Логики/LOGIC-007_Запрос-push-разрешения.md) в будущем).

## 7. Не входит в MVP (Phase 2+)

| Функция | Причина / источник |
|---------|--------------------|
| **Онлайн-оплата** | На старте оплата офлайн (BR-8, FR-30); онлайн — за рамками скоупа. |
| **Авто-отмена по погоде + прогноз** | Владелец/маршал отменяют вручную, приложение только уведомляет; авто-прогноз и автоматическая отмена — Phase 2. |
| **Полноценная программа лояльности** | Бейджи, скидки, приоритетная запись, персональные предложения для постоянных клиентов — не в MVP (NFR-16). В MVP — только индикатор «Постоянный клиент» в профиле (если данные есть в API). |
| **Публичные отзывы / рейтинги маршалов для всех клиентов** | Агрегаты и детальные отзывы видит только админ в своей панели; публичный показ — позже. |
| **Мобильные интерфейсы маршала и владельца** | Эти роли работают через существующую инфраструктуру/админку (R-028). |
| **История всех заездов клиента с фото/видео** | Только базовый список + детали в MVP. |
| **Интеграция с Apple Wallet / Google Pay для билета** | Phase 2 (после онлайн-оплаты). |

## 9. Замечания по данным

- **Оценка маршала** — включается в MVP (явный запрос заказчика в брифе). В отличие от «Волны», здесь это приоритетная функция для мотивации инструкторов.
- **API** описан в многофайловой OpenAPI-спецификации (точка входа — `redocly.yaml` или аналог). Домены: **auth** (запрос/проверка кода, logout), **profile** (профиль, смена телефона, удаление, aggregate постоянного клиента), **slots** / **rides** (список/фильтрация, карточка, read-only), **bookings** (создание, список, отмена, оценка), **marshals** (справочник + возможно оценки, read-only), **gear** или inline в slots (тариф проката, свободный фонд).
- Существующий бэкенд — **black-box источник истины** (R-004, R-015). Приложение не создаёт/редактирует расписание, не управляет маршалами и фондом — только потребляет и бронирует через контракт API.
- **Нет исторических данных** (учебный/тестовый проект) — миграция и дозаполнение вне скоупа.
- **Push-токен** хранится в защищённом хранилище; регистрация/обновление — при авторизации и по событиям (LOGIC-007).
- Числа (capacities, free counts, цены, тарифы) **не хардкодятся** — всегда из ответа API.
