## 1. Назначение и контекст

Экран показывает **все параметры одного заезда (слота)**, чтобы клиент мог принять решение
о записи и перейти к оформлению. Это промежуточный шаг между списком слотов
([SCR-002](SCR-002-slot-list.md)) и оформлением записи ([SCR-004](SCR-004-booking.md)) —
вложенный экран без таб-бара (см. [foundations §4.2](00-foundations.md#42-таб-бар-авторизованная-зона)).

Контекст использования — уличный картинг-центр, активный отдых, клиенты ценят время и
определенность ([brief-karting.md](brief-karting.md)): клиенту нужно быстро считать
ключевые параметры (когда, какая трасса, кто инструктор, есть ли карты) и решить, записываться ли.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** убедиться, что заезд подходит (время, конфигурация трассы, инструктор),
  и понять, есть ли свободные карты, чтобы записаться.
- **Задача экрана:** дать полную и честную картину слота за один экран и предоставить
  один очевидный следующий шаг — «Записаться» — либо понятно объяснить, почему запись
  сейчас невозможна.
- **User story:** US-4 — «Как клиент, я хочу видеть все детали заезда перед записью,
  чтобы выбрать подходящий слот».

---

## 3. Навигация

**Входящие:**
- Тап по карточке заезда в списке [SCR-002](SCR-002-slot-list.md).

**Исходящие:**
- **«Записаться»** (основной CTA) → [SCR-004 Оформление записи](SCR-004-booking.md).
  Активен только при наличии свободных мест (см. §6, §8).
- **Тап по карте трассы / «Открыть карту»** → шторка [BS-004 «Карта трассы»](BS-004-route-map.md)
  (открытая интерактивная карта с отрисованной конфигурацией).
- **Назад** (в хедере / системный жест) → возврат к списку [SCR-002](SCR-002-slot-list.md).

Полная карта переходов — в [фича-листе §3](../5-mobile-app-spec/feature-list.md#3-карта-навигации).

## 4. Контент-инвентарь

Все значения приходят из данных слота/трассы/инструктора (read-only API). Лимиты и числа
**не хардкодятся** — подставляются из данных
(см. [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).

| # | Поле (модель данных) | Что показываем | Обяз. | Примечание |
|---|----------------------|----------------|:-----:|------------|
| 1 | `slot.start_at` | Дата и время старта | Да | Крупно, контрастно (ключевое число). Дата + время. |
| 2 | `track.name` | Конфигурация трассы | Да | Заголовок параметра «Трасса». |
| 3 | `track.difficulty` | Сложность (новичковая / опытная) | Да | Бейдж/подпись рядом с названием. |
| 4 | `track.duration_min` | Длительность заезда | Нет | Показывать, **если значение есть** в данных; при отсутствии блок скрывается. |
| 5 | `instructor.name` | Имя маршала-инструктора | Да | Без контактов инструктора ([foundations §8.2](00-foundations.md#82-безопасность-данных-в-ui-nfr-11-nfr-12)). |
| 6 | `slot.price` | Цена | Да | Цена за одного пилота (per-person). Крупно, контрастно. |
| 7 | `slot.total_seats` / `slot.free_seats` | Всего / свободно картов | Да | Свободно — ключевое число, влияет на состояние CTA. |
| 8 | `slot.free_gear_sets` | Свободные комплекты экипировки | Да | Число свободных комплектов (из данных, без хардкода). |
| 9 | `track.geometry` | Карта трассы | Да | **Статичный превью Яндекс-карты с выделенной трассой** (компонент [foundations §4.5](00-foundations.md#45-карта-маршрута-статичный-превью-яндекскарт)). |
| 10 | `slot.meeting_point` (+ координаты) | Место сбора | Да | **Обязательный элемент.** Пин на карте + текст под картой. |

> Доступные для записи места рассчитываются ниже по сценарию как
> `min(свободные карты, потолок трассы)`; на этом экране показываем
> фактические `free_seats` слота.

## 5. Структура и иерархия

Каркас — по [foundations §4.1](00-foundations.md#41-базовый-каркас): фиксированный хедер,
скролл-контент, фиксированный нижний CTA. Таб-бар скрыт.

- **Хедер (фикс.):** кнопка «Назад» + заголовок «Карточка заезда».
- **Скролл-контент:** блоки параметров слота (§4) сверху вниз по приоритету:
  дата/время → трасса+сложность (+длительность) → карта трассы + место сбора →
  маршал → доступные карты → экипировка → цена.
- **Нижний CTA (фикс.):** кнопка «Записаться» во всю ширину. Рядом — строка цены/доступности.

```

│                                  │
│  ┌─────────────────────────────┐ │

│  │ Длинная конфигурация        │ │  ← track.name

│  └─────────────────────────────┘ │
```

## 6. Компоненты и поведение

| Компонент | Поведение |
|-----------|-----------|
| **Кнопка «Назад»** | Возврат на [SCR-002](SCR-002-slot-list.md). |
| **Блоки параметров** | Лейбл + значение. Крупные и контрастные числа (дата, цена, свободные карты). |
| **Блок «Карта трассы»** | Статичный превью. Тап → шторка [BS-004](BS-004-route-map.md). |
| **CTA «Записаться»** | При `free_seats > 0` — активна. При `free_seats = 0` — неактивна, текст «Мест нет». |

---

## 7. Состояния экрана

- **Loading:** скелетон блоков.
- **Content:** все параметры (§4) + активный CTA.
- **Error:** заглушка с кнопкой «Обновить».
- **Нет свободных мест:** CTA «Мест нет» (неактивна).
- **Слот отменён (по погоде):** пометка «Заезд отменён из-за погодных условий»; CTA скрыт или неактивен.

---

## 8. Валидации и микрокопия

- **Доступность мест:** «Свободно `{free_seats}` из `{total_seats}`».
- **Экипировка:** показывается число доступных комплектов (`slot.free_gear_sets`).
- **Напоминание об оплате:** «Оплата на месте: наличные или перевод на карту.»
- **Карта:** обязательный пин места сбора; если карта не грузится — fallback на текстовое описание.

---

## 9. Критерии приёмки дизайна

- **AC-001:** Отображение всех параметров заезда (дата, трасса, инструктор, цена, карты, экипировка, карта).
- **AC-002:** При `free_seats > 0` — переход на [SCR-004](SCR-004-booking.md).
- **AC-003:** При `free_seats = 0` — CTA неактивна с пометкой «Мест нет».
- **AC-004:** Отменённый заезд (погода) имеет статус и заблокированный CTA.
- **AC-005:** Карта трассы корректно открывает шторку [BS-004](BS-004-route-map.md).

## 10. Критерии приёмки дизайна

```gherkin
AC-001: Отображение всех параметров заезда
  Дано клиент открыл карточку заезда со свободными местами
  Тогда на экране показаны дата и время старта, название конфигурации и сложность трассы,
    имя маршала-инструктора, цена за заезд (пилота), всего и свободно картов и доступность комплектов экипировки
  И длительность заезда показана, если значение есть в данных
  И показаны карта трассы и место сбора

AC-008: Карта трассы и место сбора
  Дано клиент открыл карточку заезда
  Тогда показан статичный превью Яндекс-карты с выделенной линией конфигурации трассы и пином места сбора
  И под картой показан текстовый блок места сбора (адрес/бокс/ориентир)
  Когда клиент тапает по карте или «Открыть карту»
  Тогда открывается шторка BS-004 с интерактивной картой и отрисованным маршрутом трассы

AC-009: Карта недоступна — текстовый fallback
  Дано карта не загрузилась (ошибка/офлайн/нет ключа)
  Тогда вместо карты показан текстовый блок места сбора и ссылка «Открыть в Яндекс.Картах»
  И остальные параметры заезда и кнопка «Записаться» остаются доступны

AC-002: Переход к оформлению при наличии мест
  Дано заезд имеет free_seats больше нуля
  Тогда кнопка «Записаться» активна
  Когда клиент нажимает «Записаться»
  Тогда происходит переход на экран оформления записи (SCR-004)

AC-003: CTA неактивна при отсутствии мест
  Дано заезд имеет free_seats равное нулю
  Тогда кнопка «Записаться» неактивна
  И рядом с кнопкой показана пометка «Мест нет»
  И тап по кнопке не выполняет переход

AC-004: Возврат к списку
  Дано клиент находится на карточке заезда
  Когда клиент нажимает «Назад»
  Тогда происходит возврат к списку заездов (SCR-002)

AC-005: Состояние ошибки загрузки
  Дано запрос данных заезда завершился ошибкой
  Тогда показана заглушка ошибки и кнопка «Обновить»
  Когда клиент нажимает «Обновить»
  Тогда выполняется повторный запрос данных заезда

AC-006: Отменённый или недоступный слот (заезд)
  Дано заезд отменён центром или недоступен для записи (например, из-за погоды)
  Тогда показаны параметры заезда с явной пометкой о причине («Заезд отменён из-за погодных условий»)
  И кнопка «Записаться» неактивна или скрыта, а повторная запись на слот запрещена

AC-007: Напоминание об офлайн-оплате
  Дано клиент видит блок цены
  Тогда показано напоминание «Оплата на месте: наличные или перевод на карту.»
```

## 11. Решения

- **Длительность заезда:** показывается **при наличии** `track.duration_min`; при отсутствии блок скрывается (§4, §6).
- **Комплекты экипировки:** показываются **числом** свободных комплектов экипировки (`slot.free_gear_sets`) (§6, §8). Клиент на следующем экране сможет указать, нужна ли ему экипировка центра.
- **Описание конфигурации трассы:** **вне MVP** — отдельного текстового блока описания нет; на карточке отображаются название, тип/сложность и карта.
- **Место сбора:** **обязательный элемент** — пин на карте трассы + текстовый блок (бокс/ориентир, `slot.meeting_point`). См. `data-model`.
- **Конфигурация трассы — карта:** статичный превью Яндекс-карты с выделенной линией конфигурации (`track.geometry`); тап → шторка [BS-004](BS-004-route-map.md) с интерактивной картой. Компонент — `foundations §4.5`. Ключ API Яндекс.Карт — параметр конфигурации бэкенда.
