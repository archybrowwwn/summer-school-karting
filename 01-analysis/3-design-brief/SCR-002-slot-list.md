## 1. Назначение и контекст

**SCR-002 «Список заездов» — главный экран приложения** и стартовая вкладка авторизованной
зоны. Показывает каталог доступных заездов на картинге и служит **единственной точкой
входа в сценарий записи**.

По умолчанию отображаются слоты на **ближайшие 7 дней** (дефолт API; больший период — фильтром
дат), отсортированные по ближайшей дате старта и сгруппированные по дням; заполненные слоты
показываются с пометкой **«Мест нет»** (см. §6.5). Фильтр «только со свободными местами»
доступен в [BS-001](BS-001-filters.md), но **по умолчанию выключен**. Отсюда клиент открывает
карточку заезда для записи или уточняет выдачу через фильтры.

Экран работает с API в режиме read-only (см. [`00-foundations.md` §1](00-foundations.md)) и
подчиняется сквозному паттерну состояний ([`00-foundations.md` §5](00-foundations.md)).

---

## 2. Пользователь и цель

**Роль — Клиент.** Контекст использования — уличная площадка, нередко яркое солнце, спешка,
желание быстро занять место (см. [`00-foundations.md` §1](00-foundations.md)).

**Цель на экране:** быстро просмотреть ближайшие заезды и за минимум шагов выбрать подходящий
(дата, конфигурация трассы, инструктор, цена, есть ли места). Список оптимизирован под
**быстрый выбор**: крупные карточки, контрастные ключевые числа, минимум мелкого текста (принципы
P1, P2, P4 в [`00-foundations.md` §2](00-foundations.md)).

## 3. Навигация

### Входящие переходы
- После успешного входа/регистрации с [SCR-001](SCR-001-registration.md) — это стартовый экран АЗ.
- По тапу на таб **«Заезды»** в таб-баре (с любого корневого экрана вкладок).
- Возврат с карточки заезда [SCR-003](SCR-003-slot-card.md) (кнопка «назад») и из сценария записи.
- Закрытие шторки фильтров [BS-001](BS-001-filters.md) (Применить / Сбросить / закрытие).

### Исходящие переходы
- Тап по карточке заезда → карточка заезда [SCR-003](SCR-003-slot-card.md).
- Тап по кнопке/иконке **«Фильтры»** → шторка [BS-001](BS-001-filters.md).
- Таб **«Мои записи»** → [SCR-005](SCR-005-my-bookings.md).
- Таб **«Профиль»** → [SCR-007](SCR-007-profile.md).

Таб-бар (три раздела, подписи под иконками, поведение на вложенных экранах) описан в
[`00-foundations.md` §4.2](00-foundations.md) — **ссылаемся, не переописываем**.

## 4. Контент-инвентарь

Каждая **карточка заезда** агрегирует данные слота, конфигурации трассы и маршала
(см. [модель данных](../4-design/data-model.md)):

| Элемент | Источник данных | Назначение в UI |
|---------|------------------|------------------|
| Дата и время старта | `slot.start_at` | Когда заезд; ключевое число (контрастно). |
| Конфигурация трассы | `route.name` | Что за трасса (короткая / длинная). |
| Уровень трассы | `route.type` | Новичковый / Опытный; читаемый лейбл. |
| Маршал | `instructor.name` | Кто ведет инструктаж. |
| Цена | `slot.price` | Стоимость заезда; ключевое число (контрастно). |
| Всего мест | `slot.total_seats` | Лимит группы (M в формулировке «N из M»). |
| Свободно мест | `slot.free_seats` | Доступность; ключевое число (контрастно). |

> **Без хардкода лимитов.** Свободные места — данные слота; по сквозному правилу
> ([`00-foundations.md` §6](00-foundations.md)) доступность мест считается как
> `min(free_seats, route.capacity_cap, 5)`. Конкретные числа подставляются из данных,
> в макет не зашиваются.

**Уровень экрана (не карточки):**
- Заголовок экрана.
- Кнопка/иконка **«Фильтры»** + **индикатор активных фильтров** (видим, когда применён хотя бы
  один фильтр; см. [`00-foundations.md` §3.5](00-foundations.md)).

## 5. Структура и иерархия

Каркас следует базовому ([`00-foundations.md` §4.1](00-foundations.md)): фиксированный хедер →
скролл-список → таб-бар. Нижнего CTA на этом экране нет.

```
┌─────────────────────────────────┐
│ Заезды           [⚙ Фильтры •]  │  ← хедер: заголовок + кнопка фильтров
│                          ▲ индикатор активных фильтров
├─────────────────────────────────┤
│ ┌─────────────────────────────┐ │
│ │ [карточка заезда — см. ниже]│ │
│ └─────────────────────────────┘ │  ← скролл-список карточек
│ ┌─────────────────────────────┐ │
│ │ [карточка заезда]           │ │
│ └─────────────────────────────┘ │
│              ⋮ (скролл)          │
├─────────────────────────────────┤
│ [Заезды]  [Мои записи] [Профиль]│  ← таб-бар (foundations §4.2)
└─────────────────────────────────┘
```

**Карточка заезда (вся область — одна тап-зона):**

```
┌───────────────────────────────────────┐
│  Сб, 4 июля · 18:20                   │  ← start_at (крупно, контрастно)
│  Длинная трасса · Опытный             │  ← конфигурация · сложность
│  Маршал: Алексей                      │  ← instructor.name
│  ─────────────────────────────────────  │
│  1200 ₽            Свободно 5 из 14      │  ← base_price | free_seats из total_seats
└───────────────────────────────────────┘
```

Иерархия в карточке: дата/время старта (заголовок карточки) → конфигурация+сложность → маршал →
строка «цена / свободно картов». Уровни — по `00-foundations.md` §3.3.
Значения-примеры в wireframe условны.

## 6. Компоненты и поведение

### 6.1 Карточка заезда
- **Вся карточка — единая тап-зона** (≥ минимальной тач-зоны, `00-foundations.md` §3.1),
  по тапу — переход в карточку заезда [SCR-003](SCR-003-slot-card.md). Внутри карточки нет конкурирующих кликов.
- Видимая обратная связь на тап (press-state), фокус для screen reader (`00-foundations.md` §7).
- Карточки разделены явными отступами/границами (`00-foundations.md` §3.4).

### 6.2 Кнопка/иконка «Фильтры»
- Размещена в хедере; открывает шторку фильтров [BS-001](BS-001-filters.md).
- **Индикатор активных фильтров** — виден, когда применён ≥ 1 фильтр (точка/бейдж/счётчик);
  считывается с первого взгляда и дублируется не только цветом (`00-foundations.md` §3.5).
- Тач-зона ≥ минимальной; подпись/доступное имя обязательны.

### 6.3 Сортировка и группировка
- Заезды сортируются **по ближайшей дате/времени старта** (раньше — выше).
- При длинном списке — **заголовки-секции по дням** (sticky-разделители «дата»), чтобы клиент
  быстро ориентировался в расписании центра. Заголовок дня содержит текст.

### 6.4 Обновление доступности (pull-to-refresh)
- Жест «потянуть вниз» обновляет список и **актуальное количество свободных картов** на бэкенде. Во время обновления —
  индикатор обновления; контент не очищается без необходимости (воспринимаемая скорость, P4).

### 6.5 Подгрузка списка
- В MVP список предстоящих слотов заездов **грузится целиком** на 7 дней вперёд (R-027). Горизонт расписания ограничен, специальная пагинация не нужна.
- Если объём заездов вырастет, на API закладывается **курсорная пагинация** с догрузкой по достижению конца списка. Включается без изменения макета.

### 6.6 Слот без свободных мест
- Слот с `free_seats = 0` **показывается** с явной пометкой **«Мест нет»** (текст + иконка/форма,
  не только цвет) и **некликабелен** — не открывает карточку и сценарий записи.
- Скрыть заполненные заезды можно фильтром в [BS-001](BS-001-filters.md) (по умолчанию **выключен**).

## 7. Состояния экрана

Применяется сквозной паттерн (`00-foundations.md` §5). Специфика экрана:

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Loading** | Скелетоны карточек заездов (форма будущих карточек), не пустой экран. |
| **Content** | Список карточек предстоящих заездов (со свободными и заполненными — последние с пометкой «Мест нет»), сортировка по времени старта, секции по дням. |
| **Empty (нет расписания)** | Заглушка (R-027): «Пока нет доступных заездов» + подсказка зайти позже или обновить. |
| **Empty (нет заездов по фильтрам)** | Заглушка с подсказкой **изменить или сбросить фильтры** + быстрый доступ к [BS-001](BS-001-filters.md) (UC-3). |
| **Error** | Заглушка ошибки + кнопка **«Обновить»** (общий текст сетевой ошибки — `00-foundations.md` §6). |

**Скорость отклика:** загрузка/обновление списка ощущается **< 2–3 с** (NFR-6, принцип P4);
до получения данных показываются скелетоны, а не пустой экран.

## 8. Валидации и микрокопия

- **Счётчик свободных картов отражает актуальные данные** — обновляется по
  pull-to-refresh и при возврате на экран; не кэшируется в устаревшем виде для исключения двойных броней на стороне интерфейса.
- Формулировка доступности — **«Свободно N из M»**, где N = `free_seats`, M = `total_seats` (например: «Свободно 4 из 8» для новичковой трассы или «Свободно 10 из 14» для длинной трассы).
- Заезд без мест — пометка **«Мест нет»** (текст + иконка/форма, не только цвет).
- Лейбл сложности — **«Новичковый» / «Опытный»** (мапится из `route.type`).
- Цена и время — из данных слота; валюта и формат едины по приложению (знак ₽ после числа).
- Тексты Empty States:
  - нет заездов вообще — «Пока нет доступных заездов. Загляните позже.»;
  - нет по фильтрам — «По выбранным фильтрам ничего не нашлось. Измените или сбросьте фильтры.»
- Тон — простой, технический, без давления (`00-foundations.md` §6).

---

## 9. Доступность и mobile-first

- Все сквозные правила — `00-foundations.md` §7 и §3.1–3.2.
- **Крупные карточки** и единая тап-зона на заезд — критично для использования прямо у трассы в экипировке, «мокрыми или холодными пальцами» (из брифа).
- **Ключевые числа контрастны и крупны**: количество свободных картов, базовая цена, время старта (§3.2 foundations).
- Сложность трассы, индикатор фильтров и пометка «Мест нет» — передаются не только цветом, но текстом/иконкой для работы под ярким солнцем.
- Поддержка системного увеличения шрифта: карточка динамически растягивается и не ломается при крупном тексте.
- Карточки и кнопка «Фильтры» имеют доступные имена для программы чтения с экрана.

## 10. Критерии приёмки дизайна

```gherkin
AC-001: По умолчанию — предстоящие слоты со свободными картами
  Дано клиент авторизован и открыл вкладку «Заезды»
  И фильтры не применены
  Когда список заездов загружен
  Тогда показаны предстоящие слоты на ближайшие 7 дней
  И каждая карточка содержит время старта, конфигурацию трассы и её сложность,
    имя маршала-инструктора, базовую цену и статус «Свободно N из M»

AC-002: Переход в карточку заезда по тапу
  Дано на экране отображается карточка заезда со свободными местами
  Когда клиент тапает в любую область карточки заезда
  Тогда открывается карточка заезда SCR-003 для этого слота

AC-003: Открытие фильтров и индикатор активных фильтров
  Дано клиент находится на экране списка заездов
  Когда клиент тапает на иконку «Фильтры»
  Тогда открывается шторка фильтрации BS-001
  И после применения хотя бы одного фильтра в хедере появляется индикатор активных фильтров

AC-004: Пустой результат по фильтрам
  Дано применены фильтры в шторке
  Когда на бэкенде по ним нет ни одного доступного заезда
  Тогда на экране списка показан empty state с подсказкой изменить или сбросить фильтры
  И доступна кнопка быстрого сброса/перехода к фильтрам

AC-005: Обновление по pull-to-refresh
  Дано список заездов отображается на экране
  Когда клиент тянет список вниз (pull-to-refresh)
  Тогда список и актуальные счётчики свободных картов обновляются из API
  И во время обновления на экране виден лоадер/индикатор обновления

AC-006: Состояние загрузки — скелетоны
  Дано экран открывается впервые и данные от бэкенда ещё не получены
  Когда выполняется первичная загрузка данных
  Тогда на экране показываются скелетоны карточек заездов, повторяющие их форму
  И время отклика до появления контента укладывается в 2–3 секунды (NFR-6)

AC-007: Ошибка загрузки списка
  Дано запрос списка заездов к API завершился сетевой или серверной ошибкой
  Когда отображается экран
  Тогда вместо списка показана заглушка ошибки с понятным текстом и кнопкой «Обновить»
  И повторный тап по кнопке «Обновить» возвращает экран в состояние Loading

AC-008: Слот без свободных картов
  Дано у заезда наступил лимит бронирований (free_seats = 0)
  И фильтр «только со свободными местами» выключен по умолчанию
  Тогда этот заезд отображается в общем списке с пометкой «Мест нет»
  И карточка становится некликабельной (не открывает экран SCR-013)
  И при включении фильтра «только со свободными местами» такая карточка исчезает из выдачи

AC-009: Сортировка и секционирование по дням
  Дано список содержит расписание заездов на разные даты недели
  Когда список успешно загружен на экран
  Тогда карточки строго отсортированы по возрастанию даты/времени старта
  И список разделен визуальными текстовыми заголовками-секциями по дням недели
```

## 11. Решения и параметры конфигурации

| # | Решение |
|---|---------|
| 1 | **Сортировка по умолчанию** — хронологическая по ближайшей дате и времени старта заезда; заголовки-секции по дням недели для улучшения сканирования экрана (§6.3). |
| 2 | **Заполненные заезды** — отображаются в общем расписании с плашкой «Мест нет» и блокировкой кликабельности, чтобы пользователи видели плотность расписания. Скрываются опциональным фильтром (§6.6). |
| 3 | **Пагинация** — отсутствует в MVP, так как глубина расписания составляет строго 7 дней (R-027). В API заложен курсорный механизм на будущее (§6.5). |
